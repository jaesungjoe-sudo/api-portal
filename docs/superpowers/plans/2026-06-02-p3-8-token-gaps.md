```ts
  // z-index scale
  if (misc.zIndex) {
    lines.push("");
    for (const [name, value] of Object.entries(misc.zIndex)) {
      lines.push(`  --z-${name}: ${value};`);
    }
  }
  // Motion (durations + easings)
  if (misc.motion?.duration) {
    lines.push("");
    for (const [name, value] of Object.entries(misc.motion.duration)) {
      lines.push(`  --duration-${name}: ${value};`);
    }
  }
  if (misc.motion?.ease) {
    lines.push("");
    for (const [name, value] of Object.entries(misc.motion.ease)) {
      lines.push(`  --ease-${name}: ${value};`);
    }

  // Focus ring width
  if (misc.ring?.width) {
    lines.push("");
    lines.push(`  --ring-width: ${misc.ring.width};`);
  }
```

찾을 위치: `lines.push("");` 다음의 `  lines.push(\`  --radius: \${misc.radius.base};\`);` 직후. 그 다음 `lines.push("}");` 가 :root 블록 종료. 신규 코드는 종료 전에 들어가야 함.

### Task 10: `rules/motion.md` 작성

**Files:**
- Create: `design-system/rules/motion.md`

- [ ] **Step 1: 필수 섹션 골격 작성**

1. **제목**: `# Motion — Duration & easing`
2. **Truth 박스**: "코드 truth. Figma 라이브러리에는 spec 페이지(one-way 참조)만 — 자동 sync 안 함."
3. **Duration 매트릭스**:

   | 토큰 | 값 | Tailwind | 사용처 |
   |---|---|---|---|
   | `--duration-fast` | 100ms | `duration-100` | Popover / Dropdown / Select / Dialog / Tooltip open·close |
   | `--duration-base` | 200ms | `duration-200` | Sheet, Sidebar 접힘·펼침, Accordion |
   | `--duration-slow` | 300ms | `duration-300` | NavigationMenu 큰 전환, hero motion |

4. **Easing 매트릭스**:

   | 토큰 | 값 | Tailwind | 사용처 |
   |---|---|---|---|
   | (default) | `linear` | `ease-linear` | Sidebar 등 단순 사이즈/위치 보간 (default) |
   | `--ease-emphasized` | `cubic-bezier(0.22, 1, 0.36, 1)` | `ease-emphasized` | NavigationMenu, "spring 느낌" 필요한 곳 |

5. **Tailwind v4 통합 설명**: Duration 은 Tailwind native (`duration-100/200/300`) 가 그대로 동치. `ease-emphasized` 는 `globals.css` `@theme inline` 등록으로 활성화.
6. **`prefers-reduced-motion` 박스**: "현재 글로벌 핸들러 없음. 후속 작업 검토 항목 — 본 문서에서 정책 합의: 사용자가 OS 에서 reduced motion 활성화 시 모든 transition 을 `0ms` 로 단축. 구현 시 `@media (prefers-reduced-motion: reduce)` 글로벌 룰을 `globals.css` 에 추가."
7. **사용 가이드** 3 항목: popup=fast, drawer/sidebar=base, nav/hero=slow.
8. **안티패턴 5 종**:
   - 리터럴 ms (`duration-[180ms]` 등)
   - 5개 이상 duration 사용 (스케일 외 확장)
   - 불일치 페어링 (예: `duration-fast` + `ease-emphasized` — emphasized 는 slow 와 매칭)
   - `prefers-reduced-motion` 무시한 항시 애니메이션
   - 200ms 와 150ms 혼용 (현재 한 곳에 150 이 남아있다면 그건 정리 대상)
9. **Cross-refs**: spec doc.

3. **토큰 표**:

   | 토큰 | 값 | 사용 |
   |---|---|---|
   | `--ring-width` | 3px | 모든 focus-visible (`ring-3`) |
   | `--ring-color-default` | `var(--ring) / 50%` | 일반 focus (구현은 `ring-ring/50`) |
   | `--ring-color-invalid` | `var(--destructive) / 20%` | aria-invalid focus (구현은 `ring-destructive/20`) |

4. **표준 cva 패턴 스니펫**:

```ts
// 표준 focus 패턴 (Input, Select, Button, Badge, Checkbox, Toggle 동일)
"focus-visible:border-ring focus-visible:ring-3 focus-visible:ring-ring/50"

// aria-invalid 자동 색 변환
"aria-invalid:border-destructive aria-invalid:ring-3 aria-invalid:ring-destructive/20"
```

5. **`outline-none` 사용 조건**: Base UI primitive 가 자체 outline 가지므로 명시적 제거 필요. 단 키보드 사용자를 위해 `focus-visible` ring 으로 대체 보장.
6. **Anti-patterns 5 종**:
   - `ring-2` 또는 `ring-[Npx]` 리터럴 (width 통일 위반)
   - `outline-none` 만 두고 ring 없음 (키보드 사용자 차단)
   - `focus:` 사용 (마우스 클릭 시에도 ring — `focus-visible:` 사용)
   - aria-invalid 색을 ring 에만 적용하고 border 누락 (시각 누설)
   - `ring-ring/50` 외 임의 alpha (`/40`, `/60` 등 — 합의 깨짐)
7. **Cross-refs**:
   - `rules/a11y.md` §3 (Focus 관리) — *행동* (autoFocus 차단, dialog 복귀) 은 거기서.
   - 본 문서는 *시각 토큰* 만.
