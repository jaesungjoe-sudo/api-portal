# a11y / 인터랙션 베이스라인

> 코드 전반에 흩어진 a11y 룰을 한 곳에 정리. 키보드 네비, 포커스 관리, ARIA 속성, 시각 피드백 (cursor / focus ring) 베이스라인. 컴포넌트별 상세는 각 컴포넌트 spec cross-ref.

---

## 1. 적용 범위

본 룰은 다음에 적용:

- 페이지 (`(dashboard)/*`, `(docs)/*`, `(api-reference)/*`, `(design-system)/*`)
- 도메인 컴포넌트 (`src/components/api-portal/*`)
- shadcn primitive 의 *사용 측면* (primitive 내부 a11y 는 shadcn 책임)

본 룰이 다루지 않는 것:
- 색 대비비 (WCAG AA 등) — 별도 룰 / Figma 검증
- 모션 / 애니메이션 (`prefers-reduced-motion`) — 별도 룰 (현재 미정의)
- 다국어 / 텍스트 방향 (RTL) — Phase1 외

## 2. 키보드 네비

### 2.1 Tab 순서

모든 interactive 요소는 Tab 으로 도달 가능해야 함. 표준 순서:

1. TopNav 메뉴 항목 → Search → Account dropdown
2. 페이지 본문: Breadcrumb → Toolbar 검색 → CTA → 테이블 행 inline 액션 → 행 ⋯ 메뉴 → Pagination
3. 다이얼로그 열림 시: 다이얼로그 내부 첫 focusable → ... → footer 버튼 → 다이얼로그 닫힘 시 트리거로 복귀

### 2.2 `focus-visible:ring` — 표준

키보드 포커스에만 visible ring 표시 (mouse focus 는 ring 안 보임):

```tsx
className="outline-none focus-visible:ring-2 focus-visible:ring-ring"
```

- `outline-none` — 브라우저 기본 outline 제거 (focus-visible 로 대체)
- `focus-visible:ring-2` — 키보드 포커스 시 2px ring
- `focus-visible:ring-ring` — ring 색 토큰 (semantic)

### 2.3 키보드 단축키

primitive 가 기본 처리:
- **Dialog / DropdownMenu / Tooltip**: Esc 닫힘
- **DropdownMenu**: ↑↓ 항목 이동, Enter 선택
- **Tabs**: ←→ 탭 이동, Home/End 처음/끝
- **Combobox / Select**: ↑↓ 옵션, Enter 선택, Esc 닫힘

→ primitive 가 처리하므로 페이지 레벨에서 `onKeyDown` 추가 구현 X.

## 3. 포커스 관리

### 3.1 autoFocus 차단 — 모든 form-dialog

다이얼로그 열림 시 어떤 입력 필드도 자동 포커스 받으면 안 됨 (Edit 다이얼로그가 열리자마자 편집 모드 진입 차단). 2-layer 방어:

**(1) sr-only focus 흡수 span** — DialogContent 바로 아래:

```tsx
<DialogContent>
  <span tabIndex={0} className="sr-only outline-none" aria-hidden="true" />
  <DialogHeader>...</DialogHeader>
  ...
</DialogContent>
```

primitive 가 첫 focusable 노드를 찾을 때 이 span 이 잡혀 *시각적 포커스 없이* 흡수됨.

**(2) 첫 input/select 에 `autoFocus={false}` 명시**:

```tsx
<Input id="..." autoFocus={false} ... />
```

방어적 — 일부 브라우저에서 sr-only 만으로 부족할 수 있음.

> 자세한 룰: `patterns/form-dialog.md` §8

### 3.2 disabled Button 의 hover 이벤트

`<Button disabled>` 는 hover/focus 이벤트 받지 못함. Tooltip 으로 사유 안내하려면 wrap:

```tsx
<Tooltip>
  <TooltipTrigger
    render={
      <span tabIndex={0} className="inline-block outline-none focus-visible:ring-2 focus-visible:ring-ring rounded-md">
        <Button disabled>Delete</Button>
      </span>
    }
  />
  <TooltipContent>...</TooltipContent>
</Tooltip>
```

> 자세한 룰: `components/tooltip.md` 의 "Disabled action reason"

### 3.3 다이얼로그 focus 복귀

다이얼로그 닫힘 시 트리거 요소로 포커스 자동 복귀 — primitive 가 처리. 추가 구현 불필요.

## 4. ARIA 속성

### 4.1 `aria-label` — 시각적 라벨 없을 때 필수

icon-only 버튼, 트리거, overlay link 등 *시각 텍스트가 없는* interactive 요소는 `aria-label` 필수.

| 케이스 | 형식 |
|---|---|
| ⋯ DropdownMenu trigger | `aria-label={`Actions for ${name}`}` |
| Avatar trigger (Account) | `aria-label="Account menu"` |
| 검색 아이콘 버튼 (TopNav) | `aria-label="Search"` |
| 다크 모드 토글 | `aria-label="Switch to dark mode"` / `"Switch to light mode"` |
| Overlay Link (TeamCard) | `aria-label={`View ${name} team`}` |
| Copy / Eye 아이콘 버튼 (ViewApiKey) | `aria-label="Copy API key"` / `"Reveal API key"` |

> 자세한 사용 예: `components/dropdown-menu.md`, `patterns/clickable-card-with-menu.md`

### 4.2 `aria-invalid` — 폼 검증

필수값 미충족 시 Input / SelectTrigger 에 적용:

```tsx
<Input aria-invalid={errors.email} ... />
<SelectTrigger aria-invalid={errors.role}>...</SelectTrigger>
```

screen reader 가 invalid 상태 인식. 시각적 Label `text-destructive` + 에러 메시지와 짝.

> 자세한 룰: `patterns/form-dialog.md` §5

### 4.3 `aria-current="page"` — 현재 페이지

Breadcrumb 의 마지막 항목 (BreadcrumbPage primitive 가 자동 부여):

```tsx
<BreadcrumbPage>API Keys</BreadcrumbPage>
// → primitive 내부에서 aria-current="page" 적용
```

primitive 사용 시 자동 — 별도 작성 불필요.

### 4.4 `aria-describedby` — Tooltip / 폼 도움말

Tooltip primitive 가 자동 처리 — TooltipTrigger 가 TooltipContent 의 id 를 자동 참조.

폼 도움말 텍스트가 있을 때 (현재 사용 사례 없음, 향후):

```tsx
<Label htmlFor="email">Email</Label>
<Input id="email" aria-describedby="email-help" />
<p id="email-help" className="text-sm text-muted-foreground">We'll never share your email.</p>
```

### 4.5 `aria-hidden` — 장식 요소

순수 시각 요소는 screen reader 에서 숨김:

| 케이스 | 적용 |
|---|---|
| Breadcrumb separator (ChevronRight) | `aria-hidden="true"` (primitive 자동) |
| 아이콘 + 텍스트 라벨 동반 (예: Settings + "Settings" 텍스트) | 아이콘에 aria-hidden — 라벨이 의미 전달 |
| sr-only focus 흡수 span | `aria-hidden="true"` — interactive 가 아닌 정보 제공 아님 |

### 4.6 role / role 분기 (Toast)

Sonner primitive 가 자동 부여:
- success / info / warning → `role="status"` (passive announce)
- error → `role="alert"` (assertive announce — 즉시 읽힘)

## 5. 시각 피드백

### 5.1 글로벌 cursor pointer (`globals.css`)

```css
button:not(:disabled):not([aria-disabled="true"]),
[role="button"]:not([aria-disabled="true"]) {
  cursor: pointer;
}
```

- 모든 클릭 가능한 button / `[role=button]` 에 hover pointer cursor
- **disabled / aria-disabled 는 제외** — 기본 (`not-allowed` 또는 `default`) 유지
- 별도 컴포넌트 레벨에서 cursor 지정 불필요 (글로벌이 처리)

### 5.2 `cursor-default` — 클릭 액션 아닌 hover-only 요소

Tooltip 트리거인 Info / Help 아이콘:

```tsx
<Info className="h-3.5 w-3.5 text-muted-foreground cursor-default" />
```

→ hover 시 툴팁만 표시되고 클릭 액션 없음을 시각적으로 명시 (pointer cursor 가 아님으로 "클릭 가능" 오해 차단).

### 5.3 `hover` + `focus-within` 페어

mouse 호버 ↔ 키보드 포커스 시각 피드백 일관:

```tsx
className="hover:bg-muted/30 focus-within:bg-muted/30"
```

- `hover:bg-muted/30` — mouse 호버
- `focus-within:bg-muted/30` — 자식 어디든 포커스 시 동일 시각

→ 키보드 사용자도 mouse 사용자와 같은 시각 피드백.

> 자세한 룰: `patterns/clickable-card-with-menu.md` §4

## 6. 컴포넌트별 a11y 체크리스트

| 컴포넌트 / 패턴 | a11y 룰 |
|---|---|
| **Button** | 글로벌 cursor-pointer (globals.css). disabled 시 자동 제외. icon-only 일 때 `aria-label` 필수. |
| **Dialog** | Esc 닫힘 (primitive). focus 복귀 (primitive). form-dialog 의 sr-only span (§3.1). |
| **DropdownMenu** | trigger `aria-label` 필수. ↑↓ 키 (primitive). align="end" 화면 우측 안전. |
| **Tooltip** | TooltipProvider 루트 mount. delay=0. disabled trigger 는 `<span tabIndex={0}>` wrap. `cursor-default` icon. |
| **Toast (Sonner)** | role="status"/"alert" (primitive 자동). 영구 정보는 toast 금지 (states.md §5). |
| **Breadcrumb** | aria-label="breadcrumb" (primitive). 마지막 항목 BreadcrumbPage = aria-current="page" 자동. |
| **Input / Select** | `aria-invalid` + Label `text-destructive` 짝. `htmlFor` ↔ `id` 매칭. |
| **Table** | SortableHead 에 정렬 가능 표시 (lucide chevron icon). Action ⋯ 의 aria-label 에 row 식별 정보 (`Actions for ${name}`). |
| **Sidebar (Sheet 모바일)** | SheetHeader 의 `<SheetTitle className="sr-only">` (a11y 라벨만, 시각 X). SidebarTrigger `aria-label`. |
| **form-dialog 패턴** | sr-only focus 흡수 span + 첫 input autoFocus={false}. |
| **clickable-card-with-menu** | overlay Link `aria-label`, focus-visible:ring, hover+focus-within 페어. |

## 7. 안티패턴

| ❌ | 이유 |
|---|---|
| icon-only 버튼 / 트리거에 `aria-label` 누락 | screen reader 가 식별 못 함. 클릭 가능한지 알 수 없음. |
| `outline-none` 만 적용 (focus-visible:ring 없이) | 키보드 사용자에게 포커스 위치 안 보임. 항상 짝으로. |
| 시각용 아이콘에 `cursor-pointer` 강제 | 클릭 가능해 보임. Info / Help 아이콘은 `cursor-default`. |
| disabled 요소에 Tooltip 트리거 직접 적용 | disabled 요소는 이벤트 받지 못함. `<span tabIndex={0}>` wrap 필수. |
| 글로벌 cursor 룰 무시하고 컴포넌트마다 `cursor-pointer` 추가 | 글로벌이 처리. 중복 — 단 disabled 분기 누락 위험. |
| `aria-invalid` 없이 시각 에러만 (`text-destructive`) | screen reader 가 에러 인식 못 함. 시각 + ARIA 짝. |
| Custom 키보드 단축키 (`onKeyDown=Esc 닫기`) primitive 이미 처리 | 중복 코드. primitive 가 더 정확. |
| 마지막 Breadcrumb 항목 BreadcrumbLink 로 (자기 자신 링크) | aria-current="page" 누락. BreadcrumbPage 사용. |
| `focus-within` 없는 `hover` 단독 | 키보드 사용자 시각 피드백 결손. 짝. |
| 폼 다이얼로그가 sr-only span 누락 + autoFocus={false} 만 | 일부 브라우저에서 첫 input 에 포커스 잡힘. 2-layer 둘 다 필요. |

## 8. 출시 전 수동 체크 (권장)

새 페이지 / 컴포넌트 작업 시 다음을 직접 확인:

### 키보드 only 확인
- [ ] Tab 키로 모든 interactive 요소에 도달
- [ ] Tab 순서가 시각적 순서와 일치
- [ ] focus-visible ring 이 모든 포커스 가능 요소에 보임
- [ ] Esc 로 다이얼로그 / 메뉴 닫기 동작
- [ ] Enter / Space 로 버튼 / 메뉴 항목 활성화

### Screen reader 빠른 확인
- [ ] icon-only 버튼이 의미 있는 라벨로 읽힘
- [ ] 폼 검증 에러가 announce 됨
- [ ] Breadcrumb 의 현재 페이지가 "current page" 로 announce 됨

### 시각 일관성
- [ ] hover / focus-within 시각 피드백 짝
- [ ] disabled 요소가 not-allowed cursor (글로벌 룰)
- [ ] Tooltip 트리거가 cursor-default (클릭 아님 명시)
